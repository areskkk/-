# PRD：南康助企宝管理后台

> 文档状态：Draft  
> 模块定位：政策库、知识库、权限、标注、运营看板、系统配置  
> 目标端：管理员桌面端  
> 参考总文档：`docs/大数据中心AI产品需求文档.md`

## 1. 背景

管理后台是南康助企宝稳定运行的基础，负责政策数据维护、政策结构化、权限配置、知识库标注、人工兜底、模型效果监控和运营看板。若后台缺失，企业端问答、申报和政府端审核都无法持续可信。

## 2. 目标用户

| 用户 | 典型角色 | 核心诉求 |
| --- | --- | --- |
| 系统管理员 | 平台运营、数字化主管部门 | 管理账号、权限、系统配置 |
| 政策管理员 | 各部门政策维护人员 | 导入政策、维护版本、配置申报条件 |
| 知识库运营人员 | 标注、质检、政策服务人员 | 处理低置信度问题，回流知识库 |
| 业务管理者 | 主管部门、领导驾驶舱使用者 | 查看办理效率、政策覆盖、企业服务效果 |

## 3. 已确认范围

| 决策项 | 管理后台写法 |
| --- | --- |
| 外部平台 | MVP 不对接亲清赣商/惠企通，政策数据通过后台导入 |
| 登录认证 | MVP 自建账号体系，预留政务 OA/统一身份认证 |
| 企业画像 | MVP 企业手填为主，后台可导入 |
| 资格预判 | 规则优先，后台需管理结构化政策条件 |
| 材料复用 | MVP 只支持本次申报内复用，不建设长期企业材料库 |
| 低置信度 | 后台支持初始阈值配置，POC 后校准 |
| MVP 试点 | 核心部门约 60 份政策、100 家企业 |
| 手机端形态 | H5 响应式 |
| 企业绑定 | Agent 对低风险申请自动通过并留痕，人工后期可修改 |

## 4. 问题定义

1. 政策文件更新快，缺少版本化管理。
2. 政策非结构化，难以直接用于资格预判和审核。
3. 部门、角色、政策库、事项、材料之间存在复杂权限边界。
4. AI 问答和预审需要持续标注、评估和优化。
5. 管理部门需要看到政策服务效果和办理堵点。

## 5. 目标与非目标

### 5.1 目标

- 建立政策库和知识库管理能力。
- 支持政策结构化、版本管理、审核发布。
- 支持部门、角色、事项、材料字段权限配置。
- 建立人工兜底和标注回流闭环。
- 管理低置信度阈值、模型路由、提示词模板。
- 建立咨询、申报、审核、补正、AI 效果的运营看板。

### 5.2 非目标

- 不在后台直接替代正式政策发布流程。
- 不绕过各部门对政策内容的审核责任。
- MVP 不建设完整数据中台。
- MVP 不建设模型训练平台。
- MVP 不建设企业长期材料库。
- MVP 不对接亲清赣商/惠企通。

## 6. 解决方案

管理后台包含七个模块：

1. 政策库管理：政策导入、版本、有效期、废止、部门归属。
2. 政策结构化：适用对象、申报条件、材料、流程、时限、限制条件。
3. 权限管理：部门、角色、用户、政策库、事项、材料字段访问权限。
4. 知识库与标注：低置信度问题、人工答案、政策引用、标注质检。
5. 模型与阈值配置：模型路由、提示词模板、低置信度阈值、降级策略。
6. 运营看板：咨询、申报、审核、补正、办理时长、企业覆盖。
7. 审计日志：政策、权限、模型配置、人工标注和审批操作留痕。

## 7. 用户流程

### 7.1 政策入库

1. 政策管理员上传政策文件。
2. 系统解析政策文本并生成结构化草稿。
3. 政策管理员校对适用对象、条件、材料、流程、时限。
4. 选择部门归属、权限范围、生效日期。
5. 提交审核并发布到知识库。

### 7.2 政策结构化

1. 管理员打开政策原文和结构化字段并排页面。
2. 系统建议提取条件、材料和办理流程。
3. 管理员确认字段类型、比较符、目标值、证据类型、是否必填。
4. 结构化结果进入审核发布流程。

### 7.3 标注回流

1. 低置信度问题进入人工兜底队列。
2. 运营人员查看用户问题、检索结果、AI 答案。
3. 人工补充正确答案和政策依据。
4. 质检通过后回流知识库。
5. 系统更新检索样本和评测集。

### 7.4 权限配置

1. 管理员维护部门、角色、用户。
2. 配置角色可访问的政策库、事项、材料字段。
3. 配置审批人员可处理的事项范围。
4. 权限变更写入审计日志。

### 7.5 Prompt 模板管理

1. 管理员选择 Agent 类型，查看当前 stable 版本、draft 版本、灰度版本和历史版本。
2. 编辑 Prompt 时必须填写版本说明、适用 Agent、输入输出 Schema、工具调用约束。
3. 点击“测试”后选择固定评测样例或上传临时样例，系统展示 JSON 输出、引用、工具调用、耗时和错误。
4. 测试通过后可提交发布，支持按用户、部门、场景或流量比例灰度。
5. 灰度期间监控格式错误率、人工转接率、引用率、人工采纳率和成本。
6. 异常时一键回滚到上一 stable 版本，回滚动作写入审计。

### 7.6 模型路由配置

1. 管理员按任务类型配置候选模型、endpoint、部署位置、超时、成本上限和降级策略。
2. 敏感任务默认选择私有化或专有云模型，公开政策问答可选择低成本兼容模型。
3. 模型定版必须关联 `docs/模型POC结论与定版流转单.md` 中的 POC 指标和审批记录。
4. 配置发布前必须执行连通性测试和样例评测。
5. 变更支持灰度和回滚，并记录到 AuditLog。

## 9. 数据模型

| 实体 | 管理后台字段 | 要求 |
| --- | --- | --- |
| Policy | policy_id、title、department_id、level、status、version、effective_date、expire_date、source_url | 支持版本和废止 |
| PolicyCondition | condition_id、policy_id、field_key、operator、target_value、required、evidence_type、weight | 资格预判规则核心 |
| PolicyMaterialRequirement | requirement_id、policy_id、material_type、required、validity_days、template_url、reuse_allowed_in_application | 只标记本次申报内复用 |
| Department | department_id、name、parent_id、scope | 权限归属 |
| User | user_id、name、org_id、role、auth_provider、status | 自建账号，预留统一认证 |
| AuditLog | log_id、actor_id、action、target_type、target_id、ip、trace_id、created_at | 全量审计 |
| ThresholdConfig | config_id、scene、metric、threshold、action、version、status | 低置信度阈值配置 |
| PromptTemplate | template_id、agent_type、version、content、status、created_by | 提示词版本管理 |

## 11. 权限与安全

| 权限对象 | 控制方式 |
| --- | --- |
| 政策库 | 按部门、角色授权 |
| 政策结构化字段 | 政策管理员可编辑，审核发布后只读 |
| 材料字段 | 按材料类型、字段密级、部门授权 |
| 模型配置 | 仅系统管理员和授权算法/运营角色可改 |
| 阈值配置 | 修改需记录版本和审批人 |
| 审计日志 | 只读，按角色授权查看 |

数据分级：

- L1 公开：政策原文、公开办事指南。
- L2 内部：结构化规则、运营指标、权限配置。
- L3 企业敏感：企业画像、申报材料、联系方式。
- L4 审批敏感：审核意见、风险标记、模型 trace。

## 12. 低置信度与人工兜底

| 场景 | 初始阈值 | 后台能力 |
| --- | --- | --- |
| 检索 Top1 相似度 | < 0.75 | 可配置阈值、查看命中样本 |
| OCR 字段置信度 | < 0.85 | 可配置字段级阈值 |
| 规则证据缺失 | 必填证据缺失 | 自动进入待补充或人工队列 |
| AI 无引用或与规则冲突 | 任意发生 | 强制转人工 |

阈值必须在 POC 后基于真实样本校准。

## 13. 前端体验要求

- 管理后台必须高密度、可扫描、可筛选、可批量操作。
- 政策结构化页面需支持原文和结构化字段并排校对。
- 权限配置需清晰展示部门、角色、政策库、事项、材料字段关系。
- 看板页面必须区分业务指标、AI 指标、办理效率指标。
- 空状态、错误状态、无权限状态、发布前校验状态必须完整。
- 关键页面设计稿、截图和布局检查可参考 Pencil MCP。

## 14. 技术选型建议

| 模块 | 开源项目参考 | 适配原因 | 注意事项 |
| --- | --- | --- | --- |
| 后台前端 | Ant Design Pro | 适合管理后台、权限菜单、表格表单密集场景 | 需定制政务/企业级视觉 |
| 设计参考与验收 | Pencil MCP | 可参考设计稿、组件层级、截图和布局检查 | 不作为运行时依赖 |
| 复杂表单 | Formily | 适合政策结构化字段、动态表单和条件联动 | 需定义政策字段 schema |
| RAG/知识库 | RAGFlow | 适合政策文件解析、知识库管理和检索增强 | 需二次开发权限过滤和版本治理 |
| Agent 编排 | LangGraph | 适合人工兜底、标注回流、长流程状态管理 | 后台配置变更需进入审计 |
| 权限引擎 | Casbin | 支持 RBAC/ABAC，适合部门和事项权限 | 需统一权限模型和审计 |
| 数据看板 | Apache Superset | 开源 BI 看板，适合运营和效能指标分析 | 涉政数据需私有化部署和权限隔离 |
| 对象存储 | MinIO | 适合政策文件、申报材料、标注附件存储 | MVP 不作为长期企业材料库 |
| OCR | PaddleOCR | 支持政策文件和材料识别 | 识别结果需人工校对流程 |
| 可观测性 | OpenTelemetry + Grafana | 支持链路追踪、指标监控、日志观测 | 模型调用 trace 需脱敏 |

## 15. 数据与指标

| 指标 | 说明 |
| --- | --- |
| 政策入库数量 | MVP 约 60 份核心部门政策 |
| 政策结构化完成率 | 已结构化政策占比 |
| 政策更新及时率 | 政策发布或变更后按时更新比例 |
| 低置信度处理时长 | 人工兜底问题平均处理时间 |

| 权限异常数 | 越权访问、权限拒绝、配置异常数量 |

## 16. 验收标准

| 类型 | 标准 |
| --- | --- |
| 政策结构化 | 每条政策至少完成适用对象、条件、材料、流程、时限结构化 |
| 规则配置 | 资格预判规则可被企业端和政府端调用 |
| 阈值配置 | 检索、OCR、规则证据、模型输出阈值可配置和留痕 |
| 权限 | 部门、角色、政策库、事项、材料字段权限通过测试 |
| 人工兜底 | 低置信度问题可进入队列、处理、质检、回流 |
| 审计 | 政策、权限、模型、阈值、标注变更全量留痕 |
| 前端 | 关键页面通过 Pencil MCP 或等价截图/布局检查 |

## 17. 风险与依赖

| 风险/依赖 | 影响 | 应对 |
| --- | --- | --- |
| 政策维护责任不清 | 知识库过期 | 明确部门责任人和更新 SLA |
| 结构化字段设计不合理 | 影响推荐和审核 | 先覆盖高频政策，逐步抽象字段 |
| 权限模型复杂 | 上线周期增加 | MVP 先做部门 + 角色 + 事项权限 |
| 标注质量不稳定 | 影响模型效果 | 增加质检和抽检机制 |
| 看板口径不统一 | 管理决策失真 | 先定义指标口径和数据来源 |
