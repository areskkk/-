# 申报状态机设计

## 1. 状态图

```text
draft
  ├─ submit ─> submitted ─> pre_reviewing ─┬─> reviewing ─┬─> approved ─> archived
  │                                         │              ├─> rejected ─> archived
  │                                         │              ├─> need_supplement ─> resubmitted ─┐
  │                                         │              └─> manual_review ─────────────────┤
  │                                         ├─> need_supplement ───────────────────────────────┘
  │                                         └─> manual_review ────────────────────────────────┘
  └─ cancel ─> archived
submitted ── withdraw ─> withdrawn ─> archived
need_supplement ── timeout ─> timeout_closed ─> archived
```

## 2. 状态定义

| 状态 | 说明 | 企业可见 | 政府可操作 |
| --- | --- | --- | --- |
| draft | 草稿 | 是 | 否 |
| submitted | 已提交 | 是 | 发起预审 |
| pre_reviewing | 预审中 | 是 | 查看预审 |
| reviewing | 审核中 | 是 | 通过、驳回、补正、复核 |
| need_supplement | 需补正 | 是 | 查看 |
| resubmitted | 已补正 | 是 | 重新审核 |
| manual_review | 人工复核中 | 是 | 人工处理 |
| approved | 已通过 | 是 | 归档 |
| rejected | 不通过 | 是 | 归档 |
| withdrawn | 已撤回 | 是 | 归档 |
| timeout_closed | 超时关闭 | 是 | 归档 |
| archived | 已归档 | 是 | 只读 |

## 3. 操作约束

- `draft` 可编辑，提交后生成企业画像快照和材料版本。
- `submitted` 后不得直接修改，只能撤回或等待审核。
- `need_supplement` 必须带补正原因、材料清单、截止时间。
- `resubmitted` 后再次进入预审或人工审核。
- 并发审核采用版本号校验，版本冲突时禁止覆盖提交。

## 4. 超时规则

| 场景 | 初始规则 |
| --- | --- |
| 企业补正超时 | 默认 5 个工作日，可按事项配置 |
| 平台运营低置信度处理 | 普通任务 1 个工作日内处理 |
| 审核任务预警 | 接近事项时限前 1 个工作日预警 |
