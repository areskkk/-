# 人工兜底工作流设计

## 1. 队列归属

低置信度人工兜底统一进入平台运营队列。平台运营按问题类型处理或转派责任部门。

## 2. 触发条件

| 触发 | 说明 |
| --- | --- |
| 检索 Top1 < 0.75 | 政策召回不可信 |
| OCR 字段 < 0.85 | 材料字段不可信 |
| 必填证据缺失 | 不能完成规则判断 |
| AI 无引用 | 不允许直接答复 |
| 政策冲突/自由裁量 | 需人工判断 |

## 3. SLA

| 类型 | 时限 |
| --- | --- |
| 普通咨询/预判 | 1 个工作日内 |
| 审核争议 | 1 个工作日内转责任部门 |
| 高风险/投诉 | 即时标记并通知负责人 |

## 4. 处理流程

1. 系统生成兜底任务。
2. 平台运营查看问题、上下文、检索结果、AI 输出。
3. 运营处理或转派部门。
4. 处理结果经质检后回流知识库。
5. 更新评测集和低置信度统计。

## 5. 质检

- 每周抽检不少于 10% 兜底处理结果。
- 错误答案必须回滚并标记原因。
- 高风险错误进入复盘。
