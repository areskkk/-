# 账号与权限模型设计

## 1. 账号体系

MVP 使用自建账号体系，预留赣服通、政务统一认证和政务 OA 对接。

| 账号类型 | 说明 |
| --- | --- |
| 企业用户 | 企业经办人、负责人、财务人员 |
| 政府用户 | 窗口人员、审批人员、部门负责人 |
| 后台用户 | 系统管理员、政策管理员、知识库运营、质检人员 |
| Agent | 作为系统主体执行预审、绑定审核、标注建议，所有动作留痕 |

## 2. 企业绑定流程

1. 用户注册账号。
2. 提交企业名称、统一社会信用代码、营业执照。
3. Agent 执行低风险审核。
4. 若统一社会信用代码格式正确、企业名称一致、营业执照/OCR 信息一致，则自动通过。
5. 不满足任一条件进入平台运营人工队列。
6. 人工可后期修改绑定状态，所有修改写入审计日志。

## 3. 企业角色

| 角色 | 权限 |
| --- | --- |
| owner | 管理企业信息、授权人员、提交申报 |
| manager | 管理画像、发起申报、查看进度 |
| operator | 填写表单、上传材料、提交补正 |
| viewer | 查看政策推荐和申报进度 |

## 4. 政府角色

| 角色 | 权限 |
| --- | --- |
| window_staff | 查看咨询、协助分流、发起人工兜底 |
| reviewer | 查看授权事项、审核、补正、确认 AI 草稿 |
| department_lead | 查看部门任务、超时预警、统计指标 |

## 5. 后台角色

| 角色 | 权限 |
| --- | --- |
| system_admin | 用户、权限、系统配置 |
| policy_admin | 政策导入、结构化、版本管理 |
| kb_operator | 低置信度处理、标注回流 |
| qa_reviewer | 标注质检、评测集维护 |

## 6. 权限矩阵

| 对象 | 企业用户 | 政府用户 | 后台用户 |
| --- | --- | --- | --- |
| 企业画像 | 本企业可见/编辑 | 授权事项可见 | 按角色可导入/维护 |
| 申报材料 | 本企业本申报可见 | 授权事项可见 | 按材料密级授权 |
| 政策结构化规则 | 只读摘要 | 审核依据只读 | 政策管理员可编辑 |
| AI 审批草稿 | 不可见内部草稿 | 可确认/修改 | 可查看 trace |
| 审计日志 | 不可见 | 仅个人/部门范围 | 按角色查看 |
