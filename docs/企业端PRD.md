# PRD：南康助企宝企业端

> 文档状态：Draft  
> 模块定位：企业政策咨询、资格预判、申报辅助、进度跟踪  
> 目标端：企业 Web + 手机端完整申报入口  
> 参考总文档：`docs/大数据中心AI产品需求文档.md`

## 1. 背景

企业端是南康助企宝面向企业的统一入口。目标是让企业快速完成三件事：知道能申报什么、知道还差什么、知道下一步怎么做。

## 2. 目标用户

| 用户 | 典型角色 | 核心诉求 |
| --- | --- | --- |
| 企业决策者 | 企业主、厂长、负责人 | 快速判断能享受哪些政策 |
| 企业办事人员 | 行政、财务、人事、项目申报人员 | 减少重复填报和材料上传 |
| 企业财务人员 | 财务、税务负责人 | 判断奖补、税费、融资、技改类政策适配情况 |

## 3. 已确认范围

| 决策项 | 企业端写法 |
| --- | --- |
| 手机端 | MVP 采用 H5 响应式，支持手机端完整申报、材料上传、多政策填报 |
| 登录认证 | MVP 自建账号体系；企业绑定由 Agent 对低风险申请自动通过并留痕，人工后期可修改；预留赣服通/统一政务认证 |
| 企业画像 | MVP 企业手填为主，后台可导入 |
| 多政策申报 | 前端合并填写，后端按政策拆成多条申报事项 |
| 资格预判 | 规则优先，AI 只解释和总结 |
| 材料复用 | 只支持本次申报内复用，不做长期材料库 |

## 4. 问题定义

1. 企业不知道政策入口在哪里，也不知道哪些政策适合自己。
2. 政策条款专业，企业难以判断是否符合条件。
3. 同一申报中多个政策可能要求相同材料，企业重复上传。
4. 企业不知道申报进度、补正要求和办理时限。
5. 手机端也要能完成申报，不能只做查询页。

## 5. 目标与非目标

### 5.1 目标

- 让企业通过自然语言快速获得政策答案。
- 根据企业画像推荐可申报政策。
- 给出规则优先的资格预判、差距提示和材料清单。
- 支持桌面端和手机端完整申报。
- 支持多政策申报、本次申报内材料复用、OCR 材料检查。
- 提供清晰进度跟踪和补正提醒。

### 5.2 非目标

- 不替代政府最终审批。
- 不承诺 AI 预判等于审批结果。
- MVP 不对接亲清赣商/惠企通。
- MVP 不做企业材料长期保存和跨事项长期复用。
- 不在企业端暴露政府内部审核规则和敏感审批信息。

## 6. 解决方案

企业端由六个核心页面组成：

1. 企业首页/工作台：政策推荐、待办、进度、补正提醒。
2. 智能政策问答：文字/语音咨询、政策解释、来源引用。
3. 政策匹配与资格预判：符合度、差距、材料清单。
4. 一站式申报：多政策勾选、合并填写、材料上传、OCR 检查。
5. 手机端申报：支持政策选择、表单填写、材料上传、提交、补正。
6. 办理进度跟踪：节点状态、补正要求、办理时限、结果通知。

## 7. 用户流程

### 7.1 政策咨询

1. 企业输入问题或语音提问。
2. 系统识别意图和缺失字段。
3. 系统检索政策并返回答案、政策来源、适用条件。
4. 信息不足时追问营收、员工数、行业、出口、技改等字段。
5. 系统按规则生成资格预判，AI 生成可读说明。

### 7.2 多政策申报

1. 企业进入政策推荐页。
2. 查看可申报政策和符合度。
3. 勾选一个或多个政策进入申报。
4. 系统合并共用字段和共用材料要求。
5. 企业补充差异化字段和缺失材料。
6. OCR 校验材料完整性、有效期、字段一致性。
7. 企业确认提交。
8. 后端按政策拆成多条申报事项进入政府端审核。

### 7.3 手机端完整申报

1. 企业在手机端查看推荐政策。
2. 选择政策并填写表单。
3. 拍照或上传材料。
4. 查看 OCR 校验结果和缺失提示。
5. 确认提交或保存草稿。
6. 后续可在手机端补正。

手机端材料上传体验要求：

- 上传入口同时支持拍照、相册选择、文件选择。
- 拍照后先进入裁剪/旋转/重拍确认，再上传。
- 上传中展示进度；网络中断时保留草稿和待上传队列。
- 同一申报内已上传材料可在多政策子项中复用，页面标记“本次已上传，可复用”。
- OCR 低置信度字段用字段级提示，不阻塞草稿保存；提交前提示需人工确认或补充。
- 补正入口复用同一上传能力，并明确显示补正原因、要求材料和截止时间。

### 7.4 补正

1. 企业收到补正提醒。
2. 查看缺失材料、错误字段、补正原因和截止时间。
3. 在线补充材料或修改信息。
4. 提交后更新状态。

## 9. 数据模型

| 实体 | 企业端使用字段 | 要求 |
| --- | --- | --- |
| EnterpriseAccount | enterprise_id、user_id、role、status | 支持一个企业多个经办人 |
| EnterpriseProfile | name、credit_code、industry、scale、revenue_range、employee_count、tax_amount、export_amount、tech_upgrade_status | MVP 企业手填为主，后台可导入 |
| Application | application_id、enterprise_id、status、submit_time、deadline_at | 企业端只展示授权企业的申报 |
| ApplicationPolicyItem | item_id、policy_id、status、score、review_result | 多政策申报按子项展示状态 |
| Material | application_id、material_type、file_id、hash、issue_date、expire_date、ocr_status | 仅绑定本次申报，不作为长期材料库 |

## 11. 材料与预判规则

### 11.1 资格预判

- 符合：必填规则条件均满足。
- 不符合：硬性条件明确不满足。
- 待补充：缺少关键字段或材料。
- 需人工复核：政策冲突、自由裁量、低置信度、证据不一致。

### 11.2 材料复用

- 只在同一次申报内复用。
- 多个政策需要同一材料时，企业只上传一次。
- 文件 hash 一致时可复用。
- 材料过期、OCR 低置信度、材料类型不确定时需重新确认。

### 11.3 低置信度

- OCR 字段置信度低于 0.85：提示人工确认。
- 检索 Top1 相似度低于 0.75：不直接给确定答案。
- AI 回答无引用或与规则冲突：不展示为正式结论。

## 12. 前端体验要求

- 达到企业级/政务级产品质感，不接受 demo 式页面。
- 企业首页首屏展示“可申报政策、待办、进度、补正”。
- 政策问答结果必须展示来源、适用条件、下一步操作。
- 资格预判结果支持字段级状态：符合、不符合、待补充、需复核。
- 手机端完整申报不得只缩放桌面页面，需适配单手操作、拍照上传、分步保存。
- 关键页面设计稿、截图和布局检查可参考 Pencil MCP。

| 模块 | 开源项目参考 | 适配原因 | 注意事项 |

## 14. 指标与验收

| 指标 | MVP 目标 |
| --- | --- |
| 政策咨询有效响应率 | >= 80% |
| 答案引用率 | >= 95% |
| 资格预判准确率 | POC 后校准，初始目标 >= 85% |
| 申报完成率 | 发起申报后成功提交比例持续跟踪 |
| 本次材料复用 | 多政策申报同材料只上传一次 |
| 手机端申报 | 手机端可完成政策选择、表单填写、材料上传、提交和补正 |
| 页面体验 | 关键页面通过 Pencil MCP 或等价截图/布局检查 |

## 15. 异常与边界

| 场景 | 处理 |
| --- | --- |
| 无匹配政策 | 展示原因、建议补充画像字段、提供人工咨询入口 |
| 政策过期或废止 | 禁止新申报；已提交事项提示人工复核中 |
| 材料过期 | 标记不可用，提示重新上传 |
| 画像变更 | 新申报使用新画像；历史申报保留旧快照 |
| 多人编辑草稿 | 保存时提示版本冲突，避免覆盖 |
| 手机端上传失败 | 保留草稿，支持重试和桌面端继续 |
| AI 无引用回答 | 不展示为正式答案，提示无法确认或转人工 |

## 16. 风险与依赖

| 风险/依赖 | 影响 | 应对 |
| --- | --- | --- |
| 手机端完整申报复杂 | MVP 工作量增加 | 先覆盖高频政策和核心表单 |
| 企业画像字段不足 | 推荐和预判不准 | 渐进式表单和追问补齐 |
| OCR 识别错误 | 材料误判 | 低置信度人工确认 |
| 政策解释误导 | 企业信任下降 | 强制引用和免责声明 |
