# 政府端前端需求文档

## 1. 文档说明

本文档用于说明政府端前端的页面范围、路由位置、页面模块和模块功能点。政府端面向审核人员、部门经办人员和复核人员，核心目标是完成申报任务查看、材料核验、条件比对、AI 预审建议查看、人工审核决策、补正通知和审计留痕。

本文档只描述政府端业务页面。统一登录、无权限页、404 页等公共能力可由公共入口统一承接，政府人员登录后进入政府端。

## 2. 政府端整体信息架构

政府端统一路由前缀：

```text
/gov
```

政府端主流程：

```text
审核任务列表 → 审核详情 → 条件比对 / 材料核验 / AI 预审 → 人工决策 → 通过 / 驳回 / 补正 / 人工复核 → 审计留痕
```

## 3. 路由总览

| 路由 | 页面名称 | 页面用途 |
| --- | --- | --- |
| `/gov/tasks` | 审核任务列表页 | 展示待审核、审核中、需补正、已完成等申报任务，支持筛选和进入详情 |
| `/gov/tasks/:itemId` | 审核详情页 | 展示企业画像、政策条件、材料、OCR、AI 建议、审核记录和决策面板 |
| `/gov/manual-review` | 人工复核队列页 | 集中处理低置信度、规则冲突、证据缺失、自由裁量等需人工复核任务 |
| `/gov/audit-logs` | 审核审计日志页 | 查询审核、预审、补正、人工修改、AI 建议采纳等留痕记录 |
| `/gov/statistics` | 部门统计页 | 展示审核耗时、补正率、通过率、超时率、AI 建议采纳率等部门指标 |
| `/gov/drafts` | AI 审批意见草稿页 | 集中查看 AI 生成的审批意见草稿及其采纳、修改、忽略状态 |
| `/gov/material-preview/:fileId` | 材料预览页 | 独立查看申报材料、OCR 字段、异常标记和下载入口 |

## 4. 通用布局与公共模块

### 4.1 政府端应用布局

适用路由：`/gov/*`

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 顶部导航栏 | 展示系统名称、当前部门、用户菜单、消息提醒、退出登录 |
| 侧边菜单 | 展示审核任务、人工复核、审计日志、部门统计等入口 |
| 部门上下文 | 根据审核人员所属部门控制可见任务和可操作范围 |
| 权限控制 | 根据角色控制任务列表、详情查看、审核决策、补正通知、审计查看等权限 |
| 全局任务提醒 | 对超时任务、待处理补正、待确认 AI 草稿进行提醒 |
| 加载与错误状态 | 统一展示加载、空状态、无权限、任务不存在、接口异常和重试入口 |

### 4.2 审核状态标签

政府端需要统一展示以下状态：

| 状态 | 政府端文案 | 页面表现 |
| --- | --- | --- |
| `submitted` | 企业已提交 | 蓝色标签，表示可发起预审 |
| `pre_reviewing` | 系统预审中 | 蓝色进度标签 |
| `reviewing` | 人工审核中 | 蓝色标签，表示可执行审核决策 |
| `need_supplement` | 已发起补正 | 橙色标签，表示等待企业补充 |
| `resubmitted` | 企业已补正 | 蓝色标签，表示可重新审核 |
| `manual_review` | 需人工复核 | 紫色标签，表示需要复核人员处理 |
| `approved` | 已通过 | 绿色标签 |
| `rejected` | 不通过 | 红色标签 |
| `timeout_closed` | 超时关闭 | 灰色标签 |

### 4.3 政府端桌面体验原则

政府端以桌面办公效率为主：

- 列表页采用高密度表格。
- 审核详情页优先使用分屏布局。
- 条件比对、材料、AI 建议和决策入口应同屏可见或快速切换。
- AI 建议不能遮蔽人工判断入口。
- 所有审核动作必须可追溯。

## 5. 页面详细需求

## 5.1 审核任务列表页

路由：

```text
/gov/tasks
```

页面目标：帮助审核人员快速查看和筛选待处理任务，识别超时风险，并进入审核详情。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 任务统计卡片 | 展示待审核、审核中、需补正、人工复核、超时风险任务数量 |
| 筛选区 | 支持按状态、政策、部门、企业名称、提交时间、超时风险筛选 |
| 搜索框 | 支持按企业名称、统一社会信用代码、申报编号搜索 |
| 任务表格 | 展示任务编号、企业名称、政策名称、当前状态、提交时间、剩余办理时长、当前处理人 |
| 状态标签 | 统一展示提交、审核中、补正、人工复核、已通过、不通过等状态 |
| 超时提醒 | 对临近截止或已超时任务展示明显标记 |
| 批量操作区 | 支持后续扩展批量确认、批量分派、批量导出 |
| 行操作 | 支持查看详情、发起预审、进入补正处理、查看审计记录 |
| 分页器 | 支持页码、每页数量和总数展示 |

交互要求：

- 默认展示当前审核人员有权限处理的任务。
- 筛选条件变化后应刷新列表并保留 URL 查询参数。
- 点击任务行或详情按钮进入 `/gov/tasks/:itemId`。
- 无任务时展示空状态和筛选重置入口。

## 5.2 审核详情页

路由：

```text
/gov/tasks/:itemId
```

页面目标：展示单个申报事项的完整审核信息，支持审核人员完成预审查看、材料核验、条件比对和人工决策。

建议布局：桌面端采用左右分屏；左侧展示企业信息、政策信息和材料，右侧展示条件比对、AI 建议和决策面板。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 任务头部信息 | 展示申报编号、企业名称、政策名称、当前状态、提交时间、办理时限、当前处理人 |
| 企业画像快照 | 展示提交时企业画像，包括行业、规模、营收、员工、纳税、出口、技改等字段 |
| 政策条件摘要 | 展示当前政策的适用对象、硬性条件、材料要求、办理流程 |
| 条件比对表 | 按条件逐项展示政策要求、企业数据、材料证据、判断结果、置信度 |
| 材料列表 | 展示材料类型、文件名、上传时间、OCR 状态、异常标记、预览入口 |
| 材料预览区 | 支持预览图片、PDF 或文档；展示 OCR 字段定位和异常高亮 |
| OCR 识别结果 | 展示识别字段、字段值、置信度、来源页码或区域、是否需要人工确认 |
| AI 预审建议 | 展示建议通过、需补充、不符合、需人工复核及理由 |
| 依据引用区 | 展示政策条款、材料字段、规则依据和 Agent 推理引用 |
| 审批意见草稿 | 展示 AI 生成的审批意见草稿，支持采纳、修改、忽略 |
| 决策面板 | 提供通过、驳回、要求补正、转人工复核等操作 |
| 补正要求表单 | 当选择要求补正时，填写补正原因、材料清单、截止时间和说明 |
| 审核记录 | 展示预审、人工审核、补正、草稿处理、最终决策等历史操作 |
| 审计 Trace | 展示 trace_id、Agent run、工具调用摘要等可追踪信息 |

交互要求：

- 发起通过、驳回、补正等操作前必须二次确认。
- 补正通知必须填写原因、材料要求和截止时间。
- AI 草稿默认不自动生效，必须人工采纳或修改后才能用于审核意见。
- 条件比对结果应使用统一文案：满足、不满足、缺材料、需人工复核。
- 审核详情页应保留人工判断入口，不能只展示 AI 结论。

## 5.3 人工复核队列页

路由：

```text
/gov/manual-review
```

页面目标：集中处理低置信度、规则冲突、证据缺失、自由裁量事项等需要人工复核的任务。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 复核任务统计 | 展示待复核、处理中、已处理、超时风险数量 |
| 筛选区 | 支持按触发原因、政策、部门、企业、时间、状态筛选 |
| 复核任务表格 | 展示企业名称、政策名称、触发原因、置信度、进入复核时间、处理人 |
| 原因标签 | 展示低置信度、AI 与规则冲突、OCR 低置信度、证据缺失、自由裁量等原因 |
| 复核详情抽屉 | 展示任务摘要、条件比对、AI 结论、材料证据和历史操作 |
| 处理面板 | 支持确认通过、要求补正、驳回、继续转交、关闭复核 |
| 回流标记 | 支持标记是否需要回流知识库、规则库或评测样例 |
| 处理记录 | 展示复核处理过程和最终结论 |

交互要求：

- 人工复核处理必须填写处理说明。
- 复核结果应同步回写审核任务状态。
- 对 AI 或规则错误导致的复核，应允许标记为后续优化样本。

## 5.4 审核审计日志页

路由：

```text
/gov/audit-logs
```

页面目标：查询政府端审核相关的操作记录，满足审批留痕和问题追溯需求。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 筛选区 | 支持按操作人、部门、操作类型、任务编号、企业名称、时间范围筛选 |
| 日志表格 | 展示操作时间、操作人、动作、目标对象、结果、trace_id |
| 操作类型标签 | 展示发起预审、审核通过、审核驳回、发起补正、采纳 AI 草稿、修改意见等类型 |
| 日志详情抽屉 | 展示操作前后状态、请求摘要、响应结果、错误信息、关联任务 |
| Trace 查看 | 支持复制 trace_id 或跳转 Agent run / 系统追踪详情 |
| 导出入口 | 支持按筛选条件导出审计记录，导出需记录操作日志 |

交互要求：

- 审计日志只读，不允许修改和删除。
- 敏感字段应按权限脱敏显示。
- 查询结果为空时展示筛选条件和重置入口。

## 5.5 部门统计页

路由：

```text
/gov/statistics
```

页面目标：为部门管理人员提供审核效率、质量和风险指标展示。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 指标概览 | 展示申报量、已办结量、平均审核耗时、补正率、通过率、超时率 |
| 趋势图 | 展示按日、周、月统计的申报量、办结量、补正量 |
| 审核耗时图 | 展示不同政策或部门的平均审核耗时 |
| 补正率图 | 展示不同政策、企业类型、材料类型的补正率 |
| AI 建议采纳率 | 展示 AI 预审建议和审批意见草稿的采纳、修改、忽略比例 |
| 人工复核率 | 展示进入人工复核的任务比例和触发原因分布 |
| 超时风险列表 | 展示临近超时或已超时的任务清单 |
| 筛选条件 | 支持按部门、政策、时间范围筛选指标 |

交互要求：

- 指标口径需在页面提供说明。
- 图表点击后可下钻到任务列表。
- 统计页仅展示当前用户有权限查看的数据范围。

## 5.6 AI 审批意见草稿页

路由：

```text
/gov/drafts
```

页面目标：集中查看 AI 生成的审批意见草稿及人工处理状态，便于审核人员复用和追踪。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 草稿筛选 | 支持按状态、政策、任务、生成时间、处理人筛选 |
| 草稿列表 | 展示草稿编号、关联任务、企业名称、政策名称、生成时间、当前状态 |
| 草稿状态 | 展示待处理、已采纳、已修改采纳、已忽略 |
| 草稿内容预览 | 展示 AI 生成的审批意见摘要 |
| 处理详情 | 查看草稿原文、人工修改内容、处理说明和处理时间 |
| 进入任务 | 跳转至关联审核详情页继续处理 |

交互要求：

- 草稿不能在列表页直接自动生效。
- 草稿处理必须关联具体审核任务。
- 已忽略草稿应保留原因，便于后续评估 AI 效果。

## 5.7 材料预览页

路由：

```text
/gov/material-preview/:fileId
```

页面目标：为审核人员提供独立材料查看能力，便于核验文件内容、OCR 结果和异常字段。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 文件基础信息 | 展示文件名、材料类型、上传时间、文件大小、所属企业、关联任务 |
| 文件预览器 | 支持图片、PDF、常见文档预览；无法预览时展示下载入口 |
| OCR 字段面板 | 展示识别字段、字段值、置信度、页码或区域位置 |
| 异常字段标记 | 标记低置信度、过期、字段缺失、格式错误、与画像不一致等异常 |
| 政策要求对照 | 展示该材料对应的政策材料要求和校验结果 |
| 审核备注 | 审核人员可填写材料核验备注，备注写入审核记录 |
| 返回任务 | 返回关联审核详情页 |

交互要求：

- 文件访问必须受任务和部门权限控制。
- 下载或预览敏感材料应记录审计日志。
- OCR 字段点击后应定位到预览区域，无法定位时展示字段来源说明。

## 6. 政府端关键交互边界

- 审核人员只能访问授权部门和授权政策范围内的任务。
- AI 预审建议只作为辅助，不能绕过人工审核决策。
- 审批意见草稿必须人工确认或修改后才能生效。
- 补正通知必须包含补正原因、材料要求和截止时间。
- 材料预览和下载必须鉴权，并记录敏感操作。
- 审核动作、AI 建议处理、补正通知和状态变更都必须写入审计。
- 低置信度、规则冲突、证据缺失和自由裁量事项应进入人工复核流程。
