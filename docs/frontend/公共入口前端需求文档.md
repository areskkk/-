# 公共入口前端需求文档

## 1. 文档说明

本文档用于说明公共入口前端的页面范围、路由位置、页面模块和模块功能点。公共入口面向所有系统用户，包括企业用户、政府审核人员、平台管理员和运营人员。公共入口负责统一登录、注册、身份识别、角色跳转、权限拦截、异常页和基础账号能力。

公共入口不承载具体业务办理逻辑，具体业务功能由企业端、政府端和后台管理端分别承接。

## 2. 公共入口定位

公共入口是整个前端系统的基础入口，主要解决以下问题：

- 用户如何登录系统。
- 企业用户如何注册账号。
- 登录后如何识别用户身份和角色。
- 不同角色如何跳转到对应业务端。
- 未登录、无权限、页面不存在、系统异常时如何展示。
- 前端如何统一保存登录态并携带 Token 请求后端。
- 用户如何退出登录、查看基础账号信息。

## 3. 适用用户

| 用户类型 | 说明 | 登录后默认去向 |
| --- | --- | --- |
| 企业用户 | 企业经办人、企业管理员 | 企业端 `/enterprise/dashboard`，未绑定企业时进入 `/enterprise/bind` |
| 政府审核人员 | 部门审核人员、复核人员 | 政府端 `/gov/tasks` |
| 平台管理员 | 系统管理员、政策管理员、运营管理员 | 后台管理 `/admin/dashboard` 或 `/admin/policies` |
| 多角色用户 | 同一账号拥有多个角色 | 身份选择页 `/select-role` |

## 4. 公共入口路由总览

| 路由 | 页面名称 | 页面用途 |
| --- | --- | --- |
| `/` | 根入口页 | 根据登录态和角色自动跳转到对应业务端 |
| `/login` | 登录页 | 用户输入账号密码登录系统 |
| `/register` | 企业注册页 | 企业用户注册账号，注册成功后进入登录或企业绑定流程 |
| `/select-role` | 身份选择页 | 多角色用户选择本次进入企业端、政府端或后台管理端 |
| `/forgot-password` | 忘记密码页 | 用户通过手机号、验证码或管理员流程重置密码 |
| `/reset-password` | 重置密码页 | 用户设置新密码，用于忘记密码或首次登录改密 |
| `/profile` | 个人账号页 | 查看当前登录用户基础信息、角色和安全设置 |
| `/403` | 无权限页 | 用户无权访问目标页面时展示 |
| `/404` | 页面不存在页 | 访问不存在路由时展示 |
| `/500` | 系统异常页 | 系统异常、接口异常或未知错误时展示 |
| `/logout` | 退出登录处理页 | 清理登录态并跳转至登录页 |

## 5. 公共入口主流程

## 5.1 未登录访问流程

```text
访问任意业务页面 → 判断无 Token → 跳转 /login → 登录成功 → 获取用户信息和角色 → 跳转目标页面或默认首页
```

功能要求：

- 未登录用户访问 `/enterprise/*`、`/gov/*`、`/admin/*` 时，应跳转至 `/login`。
- 登录页应保留原始目标地址，登录成功后优先回跳原地址。
- 如果原地址无权限，则跳转 `/403` 或对应角色默认页。

## 5.2 已登录访问根入口流程

```text
访问 / → 检查 Token → 获取用户信息 → 判断角色数量 → 单角色自动跳转 / 多角色进入 /select-role
```

功能要求：

- 企业用户默认进入企业端。
- 政府审核人员默认进入政府端。
- 管理员默认进入后台管理端。
- 多角色用户进入身份选择页。
- Token 无效或过期时清理登录态并跳转登录页。

## 5.3 多角色选择流程

```text
用户登录成功 → 检测到多个可用角色 → 进入 /select-role → 用户选择身份 → 写入当前角色上下文 → 进入对应业务端
```

功能要求：

- 用户可选择本次进入企业端、政府端或后台管理端。
- 选择后应保存当前使用角色。
- 切换角色时需重新判断目标端访问权限。

## 6. 页面详细需求

## 6.1 根入口页

路由：

```text
/
```

页面目标：作为系统默认入口，根据用户登录态和角色自动跳转。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 启动加载状态 | 展示系统加载中状态，避免白屏 |
| 登录态检查 | 检查本地是否存在 Token |
| 用户信息获取 | 调用用户信息接口，获取用户类型、角色、权限和企业绑定状态 |
| 角色判断 | 根据角色决定进入企业端、政府端、后台管理或身份选择页 |
| 异常处理 | Token 失效、用户信息获取失败时跳转登录页或异常页 |

跳转规则：

| 条件 | 跳转目标 |
| --- | --- |
| 未登录 | `/login` |
| 企业用户且未绑定企业 | `/enterprise/bind` |
| 企业用户且已绑定企业 | `/enterprise/dashboard` |
| 政府审核人员 | `/gov/tasks` |
| 平台管理员 | `/admin/dashboard` 或 `/admin/policies` |
| 多角色用户 | `/select-role` |
| 用户无任何有效角色 | `/403` |

## 6.2 登录页

路由：

```text
/login
```

页面目标：为企业用户、政府人员和管理员提供统一登录入口。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 系统品牌区 | 展示系统名称、政务/企业服务定位、系统说明 |
| 登录表单 | 输入账号、密码；支持回车提交 |
| 验证提示 | 对账号为空、密码为空、格式错误进行前端校验 |
| 登录按钮 | 提交登录请求，展示登录中状态，防止重复点击 |
| 错误提示 | 展示账号不存在、密码错误、账号停用、权限不足等错误 |
| 记住登录状态 | 根据安全策略决定是否提供记住账号或短期保持登录 |
| 忘记密码入口 | 跳转 `/forgot-password` |
| 企业注册入口 | 跳转 `/register` |
| 开发环境提示 | 开发环境可展示测试账号说明，生产环境不展示 |

交互要求：

- 登录成功后保存 Token 和用户基础信息。
- 登录成功后根据角色跳转业务端。
- 如果登录前有目标地址，优先跳转目标地址。
- 密码输入框应支持显示/隐藏密码。
- 登录失败不能清空账号输入，只清空密码或保留用户自行修改。

## 6.3 企业注册页

路由：

```text
/register
```

页面目标：企业用户创建系统账号，为后续企业绑定和申报流程做准备。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 注册说明 | 说明注册后需要绑定企业主体才能申报政策 |
| 基础账号信息 | 输入手机号、姓名、密码、确认密码 |
| 企业初始信息 | 可选输入企业名称、统一社会信用代码，注册后带入企业绑定页 |
| 手机验证码 | 支持发送验证码、倒计时、重新发送和验证码校验 |
| 密码强度提示 | 展示密码长度、复杂度和一致性校验 |
| 协议确认 | 勾选用户协议、隐私说明、数据使用说明 |
| 注册按钮 | 提交注册请求，展示注册中状态 |
| 已有账号入口 | 跳转 `/login` |

交互要求：

- 密码和确认密码必须一致。
- 手机号格式必须校验。
- 用户协议未勾选时不允许注册。
- 注册成功后可自动登录并跳转 `/enterprise/bind`，也可跳转登录页。
- 注册失败时展示明确错误原因，如手机号已注册。

## 6.4 身份选择页

路由：

```text
/select-role
```

页面目标：当用户拥有多个身份或角色时，选择本次进入的业务端。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 当前用户信息 | 展示用户姓名、手机号、所属组织或企业 |
| 身份卡片列表 | 展示可选择身份，如企业用户、政府审核人员、平台管理员 |
| 身份说明 | 说明每个身份可访问的业务范围 |
| 进入按钮 | 点击身份卡片后进入对应业务端 |
| 默认身份设置 | 可设置下次默认进入的身份 |
| 退出登录 | 不选择身份时可退出当前账号 |

交互要求：

- 只展示当前用户有权限访问的身份。
- 身份切换后应刷新当前角色上下文和菜单权限。
- 如果用户只有一个身份，不应进入该页面，应自动跳转。

## 6.5 忘记密码页

路由：

```text
/forgot-password
```

页面目标：为忘记密码的用户提供重置密码入口。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 账号输入 | 输入手机号或账号 |
| 验证码发送 | 发送短信验证码或邮箱验证码 |
| 验证码校验 | 输入验证码并校验有效性 |
| 下一步按钮 | 校验通过后进入重置密码页 |
| 返回登录 | 跳转 `/login` |
| 管理员联系提示 | 对政府人员或管理员账号，可提示联系系统管理员重置 |

交互要求：

- 验证码发送后展示倒计时。
- 验证码错误、过期、发送频繁时展示明确提示。
- 根据实际账号策略，部分角色可禁用自助找回，只展示联系管理员说明。

## 6.6 重置密码页

路由：

```text
/reset-password
```

页面目标：用户设置新密码，用于忘记密码流程、首次登录改密或管理员重置后的强制改密。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 新密码输入 | 输入新密码，支持显示/隐藏密码 |
| 确认密码输入 | 再次输入新密码 |
| 密码强度提示 | 展示密码长度、复杂度、是否包含数字和字母等规则 |
| 提交按钮 | 提交新密码 |
| 成功提示 | 重置成功后提示重新登录 |
| 失败提示 | 展示链接过期、验证码无效、密码不符合规则等错误 |

交互要求：

- 两次密码必须一致。
- 密码强度不符合规则时不允许提交。
- 重置成功后应清理旧 Token，并跳转登录页。

## 6.7 个人账号页

路由：

```text
/profile
```

页面目标：展示和维护当前登录用户的基础账号信息、安全设置和角色信息。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 基础信息 | 展示姓名、手机号、账号 ID、所属组织、创建时间 |
| 角色信息 | 展示当前用户拥有的角色和当前使用身份 |
| 安全设置 | 修改密码、退出其他设备、查看最近登录记录 |
| 当前企业 / 部门 | 企业用户展示绑定企业；政府用户展示所属部门；管理员展示管理范围 |
| 通知偏好 | 设置补正通知、审核提醒、系统告警等接收方式 |
| 退出登录 | 清理 Token 并返回登录页 |

交互要求：

- 修改密码前需验证旧密码或二次验证。
- 退出其他设备属于安全操作，应二次确认。
- 用户角色信息只读，由后台权限管理维护。

## 6.8 无权限页

路由：

```text
/403
```

页面目标：当用户已登录但无权访问目标页面时，展示无权限提示和可行动作。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 无权限提示 | 明确说明当前账号无权访问该页面或执行该操作 |
| 原因说明 | 展示可能原因，如角色不匹配、部门无权限、企业未绑定 |
| 返回入口 | 返回上一页、返回首页或进入当前角色默认首页 |
| 切换身份 | 多角色用户可跳转 `/select-role` 切换身份 |
| 联系管理员 | 展示联系管理员或申请权限说明 |

交互要求：

- 不应暴露用户无权访问的数据详情。
- 如果是企业未绑定导致，应引导到 `/enterprise/bind`。
- 如果是角色不匹配，应引导切换身份或返回默认首页。

## 6.9 页面不存在页

路由：

```text
/404
```

页面目标：当用户访问不存在的页面时，展示友好的错误提示和返回入口。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 404 提示 | 展示页面不存在或链接已失效 |
| 返回上一页 | 返回浏览器历史上一页 |
| 返回首页 | 根据当前角色返回对应默认首页 |
| 问题反馈 | 可选提供问题反馈或复制当前链接 |

交互要求：

- 404 页不应触发业务接口请求。
- 返回首页需根据当前登录用户角色判断。

## 6.10 系统异常页

路由：

```text
/500
```

页面目标：当系统出现异常、接口不可用或未知错误时，提供统一错误展示和恢复入口。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 异常提示 | 展示系统繁忙、服务异常或未知错误说明 |
| Trace 信息 | 如后端返回 trace_id，可展示并支持复制，便于排查 |
| 重试按钮 | 重新请求或刷新页面 |
| 返回首页 | 返回当前角色默认首页 |
| 联系支持 | 展示联系运维或管理员说明 |

交互要求：

- 不应展示敏感错误堆栈。
- Trace ID 可展示给用户用于反馈。
- 可恢复错误应优先提供重试入口。

## 6.11 退出登录处理页

路由：

```text
/logout
```

页面目标：统一处理退出登录逻辑。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 退出处理中 | 展示正在退出登录 |
| 清理登录态 | 清理 Token、当前用户、当前角色、当前企业上下文 |
| 跳转登录页 | 清理完成后跳转 `/login` |
| 异常兜底 | 即使服务端退出接口失败，也应清理本地登录态 |

交互要求：

- 用户主动退出时应清理所有本地敏感信息。
- Token 过期导致的退出不需要二次确认。

## 7. 公共前端基础能力需求

## 7.1 API 请求封装

功能点：

- 统一使用 `/api/v1` 作为业务接口前缀。
- 自动携带 `Authorization: Bearer <token>`。
- 统一解析后端响应包络：`success`、`data`、`error`、`trace_id`。
- 统一处理错误码：401、403、404、409、422、429、500。
- 文件上传使用 `multipart/form-data`。
- 写操作支持传入 `idempotency_key`。
- 接口错误应返回可展示给页面的错误信息。

## 7.2 登录态管理

功能点：

- 保存 Token。
- 保存当前用户基础信息。
- 保存当前使用角色。
- 保存当前企业或部门上下文。
- 支持刷新用户信息。
- 支持退出登录。
- Token 失效时自动跳转登录页。

安全要求：

- 不在前端保存明文密码。
- 退出登录时清理本地 Token 和角色上下文。
- 敏感信息不写入 URL。

## 7.3 路由守卫

功能点：

- 未登录用户访问业务页面时跳转 `/login`。
- 已登录用户访问 `/login` 时，可自动跳转默认首页。
- 企业用户访问 `/gov/*` 或 `/admin/*` 时跳转 `/403`。
- 政府审核人员访问 `/admin/*` 时跳转 `/403`。
- 管理员访问业务端页面时根据权限判断是否允许。
- 企业用户未绑定企业时，访问申报相关页面应跳转 `/enterprise/bind`。

## 7.4 角色跳转规则

| 当前角色 | 默认首页 |
| --- | --- |
| 企业用户 | `/enterprise/dashboard` |
| 政府审核人员 | `/gov/tasks` |
| 平台管理员 | `/admin/dashboard` |
| 政策管理员 | `/admin/policies` |
| 运营人员 | `/admin/dashboard` |
| 多角色用户 | `/select-role` |

## 7.5 全局错误处理

功能点：

- 401：清理登录态并跳转 `/login`。
- 403：跳转 `/403` 或在当前页面展示无权限状态。
- 404：跳转 `/404` 或展示资源不存在。
- 409：展示状态冲突、版本冲突或重复提交提示。
- 422：展示低置信度、需人工处理或校验失败提示。
- 429：展示频率限制提示。
- 500：跳转 `/500` 或展示系统异常。

## 7.6 公共 UI 组件

公共入口需要提供以下基础组件供三个业务端复用：

| 组件 | 功能点 |
| --- | --- |
| `AuthLayout` | 登录、注册、找回密码等页面的统一布局 |
| `AppLoading` | 应用启动或路由切换加载状态 |
| `ErrorResult` | 403、404、500 等错误结果展示 |
| `RoleGuard` | 角色权限控制组件 |
| `RouteGuard` | 路由登录态控制组件 |
| `UserMenu` | 用户信息、切换身份、个人中心、退出登录 |
| `TraceError` | 展示错误信息和 trace_id |
| `EmptyState` | 业务空状态展示 |
| `SubmitButton` | 防重复提交按钮 |

## 8. 公共入口关键边界

- 公共入口只负责认证、注册、角色识别、跳转和异常处理，不承载具体业务办理。
- 企业绑定属于企业端页面，但公共入口需要知道企业是否已绑定，用于跳转判断。
- 多角色用户必须明确选择当前身份，避免误入无权限业务端。
- 无权限页面不能暴露目标页面的敏感数据。
- Token 过期或无效时必须清理本地登录态。
- 所有业务端都应复用公共入口的请求封装、登录态管理、路由守卫和错误处理。
