# 后台管理前端需求文档

## 1. 文档说明

本文档用于说明后台管理端前端的页面范围、路由位置、页面模块和模块功能点。后台管理端面向平台管理员、政策管理员、运营人员、系统管理员和算法运维人员，核心目标是完成政策库维护、政策结构化、权限配置、企业绑定审核配置、审计追踪、人工兜底、模型与阈值配置、Prompt 管理和运营看板。

本文档只描述后台管理业务页面。统一登录、无权限页、404 页等公共能力可由公共入口统一承接，管理员登录后进入后台管理端。

## 2. 后台管理整体信息架构

后台管理统一路由前缀：

```text
/admin
```

后台管理主流程：

```text
政策导入 → 政策结构化 → 审核发布 → 权限配置 → 企业绑定审核配置 → 审计留痕 → 兜底与模型运营 → 看板分析
```

## 3. 路由总览

| 路由 | 页面名称 | 页面用途 |
| --- | --- | --- |
| `/admin/dashboard` | 运营看板页 | 展示业务指标、AI 指标、审核效率指标和系统运行概览 |
| `/admin/policies` | 政策库页 | 管理政策列表、政策状态、版本、导入、发布和索引同步 |
| `/admin/policies/import` | 政策导入页 | 上传政策文件、粘贴政策文本或录入政策基础信息 |
| `/admin/policies/:id` | 政策详情页 | 查看政策基础信息、原文、版本、结构化状态和发布记录 |
| `/admin/policies/:id/schema` | 政策结构化页 | 对政策条件、材料要求、流程、限制规则进行结构化编辑 |
| `/admin/permissions` | 权限管理页 | 管理部门、角色、用户、政策权限和材料字段权限 |
| `/admin/enterprise-bindings` | 企业绑定审核配置页 | 配置企业绑定自动审核规则，查看绑定申请和人工处理记录 |
| `/admin/audit-logs` | 审计日志页 | 查看政策、权限、模型配置、人工兜底、审核等操作留痕 |
| `/admin/fallback` | 人工兜底队列页 | 管理低置信度、无引用、冲突、失败等需人工处理的问题 |
| `/admin/fallback/:taskId` | 人工兜底详情页 | 处理单个人工兜底任务，编辑人工答案并标记回流 |
| `/admin/model-config` | 模型与阈值配置页 | 配置模型路由、低置信度阈值、降级策略和发布记录 |
| `/admin/prompt-templates` | Prompt 模板管理页 | 管理 Agent Prompt 模板、版本、测试、灰度、发布和回滚 |
| `/admin/agent-runs` | Agent 运行记录页 | 查看 Agent run 列表、状态、入口、耗时、错误和成本 |
| `/admin/agent-runs/:runId` | Agent 运行详情页 | 查看单次 Agent 运行的步骤、工具调用、输入输出、错误和 trace |
| `/admin/ai-dashboard` | AI 效果看板页 | 展示检索命中率、答案引用率、预判准确率、人工采纳率和成本 |
| `/admin/settings` | 系统设置页 | 管理系统基础配置、通知配置、安全配置和集成配置 |

## 4. 通用布局与公共模块

### 4.1 后台管理应用布局

适用路由：`/admin/*`

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 顶部导航栏 | 展示系统名称、当前管理员、消息提醒、环境标识、退出登录 |
| 侧边菜单 | 展示政策库、权限管理、审计日志、人工兜底、模型配置、看板等入口 |
| 面包屑导航 | 展示当前页面层级，便于从详情页返回列表页 |
| 权限控制 | 根据管理员角色控制菜单可见性、按钮可用性和字段可见性 |
| 全局搜索 | 支持搜索政策、企业、用户、Agent run、审计日志等对象 |
| 全局操作反馈 | 统一展示保存成功、发布成功、导入失败、无权限等提示 |
| 空状态与异常状态 | 统一展示无数据、无权限、接口错误、解析失败和重试入口 |

### 4.2 后台通用表格能力

适用页面：政策库、权限管理、审计日志、人工兜底、Agent 运行记录等。

功能点：

- 支持关键词搜索。
- 支持多条件筛选。
- 支持分页。
- 支持列显隐。
- 支持排序。
- 支持行操作和批量操作。
- 支持详情抽屉或跳转详情页。
- 支持导出，导出动作需记录审计。

### 4.3 后台安全与审计要求

- 政策发布、权限变更、模型配置发布、Prompt 发布、阈值调整等高风险操作必须二次确认。
- 关键配置修改必须记录操作人、操作时间、变更前后内容和 trace_id。
- 敏感字段根据角色脱敏展示。
- 审计日志只读，不允许前端提供删除或修改入口。

## 5. 页面详细需求

## 5.1 运营看板页

路由：

```text
/admin/dashboard
```

页面目标：为平台管理员和运营人员展示业务运行、审核效率和系统整体情况。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 业务指标概览 | 展示政策数量、企业数量、申报数量、办结数量、通过率、补正率 |
| 办理效率指标 | 展示平均审核耗时、超时任务数、待审核任务数、人工复核率 |
| 政策咨询指标 | 展示咨询量、有效回答率、无结果率、人工兜底量 |
| 申报趋势图 | 按日、周、月展示申报量、提交量、办结量变化 |
| 政策热度排行 | 展示咨询最多、申报最多、补正率最高的政策 |
| 部门效率排行 | 展示各部门审核数量、平均耗时、超时率 |
| 系统提醒 | 展示政策待发布、兜底任务积压、Agent 异常、索引失败等提醒 |
| 筛选条件 | 支持按时间范围、部门、政策类型筛选看板数据 |

交互要求：

- 看板图表应支持点击下钻到对应列表页。
- 指标口径应提供说明。
- 无权限查看某类指标时，应隐藏或脱敏展示。

## 5.2 政策库页

路由：

```text
/admin/policies
```

页面目标：集中管理政策列表、版本状态、结构化状态、发布状态和索引状态。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 筛选区 | 支持按关键词、部门、政策状态、生效时间、废止状态、结构化状态筛选 |
| 政策表格 | 展示政策标题、主管部门、版本、状态、生效时间、失效时间、更新时间 |
| 状态标签 | 展示草稿、待结构化、待审核、已发布、已废止、索引中、索引失败等状态 |
| 导入入口 | 跳转至 `/admin/policies/import` |
| 行操作 | 支持查看详情、编辑结构化、发布、废止、复制版本、同步索引 |
| 批量操作 | 支持批量同步索引、批量导出、批量变更状态 |
| 索引状态提示 | 展示 RAG 索引是否已同步、最后同步时间、失败原因 |
| 版本标签 | 展示当前版本号和历史版本入口 |

交互要求：

- 发布、废止、同步索引等操作必须二次确认。
- 政策状态变化应立即刷新列表。
- 索引失败应展示失败原因和重试入口。

## 5.3 政策导入页

路由：

```text
/admin/policies/import
```

页面目标：支持管理员将政策材料导入系统，形成待结构化的政策草稿。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 导入方式选择 | 支持文件上传、文本粘贴、URL 导入或 JSON 导入 |
| 文件上传区 | 上传 PDF、Word、Markdown、图片等政策材料；展示上传进度和解析状态 |
| 文本录入区 | 粘贴政策原文，支持标题、正文、附件说明分区录入 |
| 政策基础信息 | 填写政策标题、主管部门、政策级别、版本、生效时间、失效时间、来源链接 |
| 解析结果预览 | 展示系统解析出的标题、正文、附件、条款结构和异常提示 |
| 重复政策检测 | 根据标题、部门、来源链接、版本号提示可能重复的政策 |
| 保存草稿 | 创建政策草稿并跳转政策详情或结构化页面 |

交互要求：

- 导入失败时应展示失败原因并保留已填写信息。
- 大文件上传应展示进度。
- 重复政策不应直接覆盖，需用户确认创建新版本或取消。

## 5.4 政策详情页

路由：

```text
/admin/policies/:id
```

页面目标：展示单个政策的完整管理信息，包括基础信息、原文、版本、结构化和发布记录。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 政策基础信息 | 展示标题、部门、级别、版本、生效时间、失效时间、来源链接、当前状态 |
| 政策原文 | 展示政策正文、附件、条款结构，支持搜索和定位 |
| 结构化摘要 | 展示适用对象、条件数量、材料要求数量、流程配置情况 |
| 版本记录 | 展示历史版本、版本说明、创建人、发布时间和状态 |
| 发布记录 | 展示每次审核发布、废止、回滚、索引同步记录 |
| 操作区 | 支持编辑基础信息、进入结构化、发布、废止、同步索引 |
| 审计记录摘要 | 展示与该政策相关的最近操作日志 |

交互要求：

- 已发布政策的关键字段修改应创建新版本或进入待审核状态。
- 政策废止需填写废止原因。
- 进入结构化编辑时跳转 `/admin/policies/:id/schema`。

## 5.5 政策结构化页

路由：

```text
/admin/policies/:id/schema
```

页面目标：将政策原文结构化为可用于资格预判、申报表单和审核比对的条件、材料和流程配置。

建议布局：左侧政策原文，右侧结构化编辑区。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 政策原文查看器 | 展示政策原文，支持条款折叠、搜索、定位、引用选区 |
| 适用对象编辑 | 配置行业、区域、企业规模、企业类型等适用范围 |
| 条件规则编辑 | 配置营收、员工、纳税、出口、技改、资质等政策条件 |
| 条件表达式编辑 | 支持操作符、目标值、是否必填、证据类型、权重等字段 |
| 材料要求编辑 | 配置材料类型、是否必填、有效期、模板链接、本次申报内是否可复用 |
| 申报流程编辑 | 配置申报起止时间、办理部门、承诺时限、审核步骤 |
| 排除规则编辑 | 配置不允许申报或需人工复核的排除条件 |
| 表单字段预览 | 预览企业端申报表单将生成的字段和校验规则 |
| 资格预判预览 | 使用样例企业画像测试结构化规则的预判结果 |
| 保存草稿 | 保存结构化配置但不发布 |
| 发布审核 | 提交结构化结果进入发布流程 |
| 变更对比 | 对比本次修改与上一版本结构化内容差异 |

交互要求：

- 结构化字段应能关联政策原文条款。
- 保存草稿不影响线上已发布政策。
- 发布前必须校验必填条件、材料要求、申报时间和引用来源。
- 修改已发布政策结构化内容应记录版本和审计日志。

## 5.6 权限管理页

路由：

```text
/admin/permissions
```

页面目标：配置部门、角色、用户、政策库权限、审核权限和材料字段权限。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 部门树 | 展示部门层级，支持新增、编辑、停用部门 |
| 用户列表 | 展示用户姓名、账号、部门、角色、状态、最近登录时间 |
| 角色管理 | 配置管理员、政策管理员、审核员、运营人员、只读用户等角色 |
| 权限矩阵 | 按角色配置政策管理、审核任务、材料查看、模型配置、审计查看权限 |
| 政策权限 | 配置部门或角色可访问的政策范围 |
| 材料权限 | 配置不同角色可查看的材料类型和敏感字段 |
| 数据范围 | 配置用户可见部门、企业、政策和任务范围 |
| 变更确认 | 权限修改前展示变更摘要，提交后写入审计日志 |

交互要求：

- 权限变更必须二次确认。
- 禁止管理员移除自身关键管理权限时无提示。
- 材料敏感字段权限应单独展示，避免误授权。

## 5.7 企业绑定审核配置页

路由：

```text
/admin/enterprise-bindings
```

页面目标：配置企业绑定自动审核规则，并处理异常企业绑定申请。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 绑定申请列表 | 展示企业名称、统一社会信用代码、经办人、提交时间、当前状态 |
| 筛选区 | 支持按状态、企业名称、信用代码、提交时间、风险原因筛选 |
| 自动审核规则 | 配置低风险自动通过条件，如名称一致、信用代码格式正确、证照识别通过 |
| 风险原因标签 | 展示企业名不一致、证照模糊、重复绑定、信用代码异常等原因 |
| 申请详情 | 查看企业提交信息、营业执照、OCR 结果、经办人信息 |
| 人工处理 | 支持通过、驳回、要求补充材料 |
| 审核记录 | 展示自动审核和人工处理记录 |

交互要求：

- 自动审核规则变更必须记录版本和操作人。
- 驳回企业绑定申请必须填写原因。
- 企业营业执照预览和下载需鉴权并写入审计。

## 5.8 审计日志页

路由：

```text
/admin/audit-logs
```

页面目标：集中查询平台内所有关键操作留痕，满足安全、合规和问题追溯需求。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 筛选区 | 支持按操作人、角色、动作类型、目标对象、时间范围、trace_id 筛选 |
| 日志表格 | 展示操作时间、操作人、动作、目标类型、目标 ID、结果、IP、trace_id |
| 操作类型标签 | 展示政策导入、政策发布、权限变更、模型配置、Prompt 发布、人工兜底等类型 |
| 日志详情抽屉 | 展示请求摘要、变更前后、错误信息、关联对象、上下游 trace |
| 敏感字段脱敏 | 根据角色隐藏或脱敏企业敏感字段、材料字段、模型输入输出 |
| 导出功能 | 支持导出筛选结果，导出动作本身写入审计 |

交互要求：

- 审计日志只读，不提供修改和删除入口。
- 查询 trace_id 时应支持精确匹配。
- 敏感操作日志默认不展开详细内容，需要权限校验。

## 5.9 人工兜底队列页

路由：

```text
/admin/fallback
```

页面目标：管理政策问答、资格预判、审核预审等流程中进入人工兜底的任务。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 兜底任务统计 | 展示待处理、处理中、已处理、超时、重复任务数量 |
| 筛选区 | 支持按来源场景、触发原因、状态、政策、企业、时间筛选 |
| 任务表格 | 展示任务编号、来源、问题摘要、触发原因、置信度、创建时间、处理人 |
| 原因标签 | 展示无引用、低置信度、规则冲突、工具失败、用户追问不清等原因 |
| 分派操作 | 支持认领、转派、释放任务 |
| 行操作 | 支持进入详情、关闭、标记重复 |
| 批量操作 | 支持批量分派、批量关闭、批量导出 |

交互要求：

- 兜底任务认领后应显示当前处理人。
- 已处理任务应保留处理结果和回流标记。
- 重复任务应能关联原任务。

## 5.10 人工兜底详情页

路由：

```text
/admin/fallback/:taskId
```

页面目标：处理单个人工兜底任务，生成可追溯的人工答案或处理结论，并决定是否回流知识库或评测集。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 任务基础信息 | 展示任务来源、企业、政策、问题、触发原因、创建时间、trace_id |
| 原始输入 | 展示用户问题、企业画像摘要、申报上下文或审核上下文 |
| AI 原始结果 | 展示 AI 初始答案、引用、置信度、错误原因或工具失败信息 |
| 证据材料 | 展示政策条款、材料字段、规则结果和相关文件 |
| 人工答案编辑器 | 编辑人工回复、处理说明、引用依据和下一步建议 |
| 质量检查 | 检查是否有引用、是否包含确定性误导、是否缺少必要说明 |
| 回流设置 | 标记是否回流知识库、规则库、Prompt 样例或评测集 |
| 处理操作 | 支持保存草稿、提交处理、关闭任务、转派任务 |
| 处理历史 | 展示任务处理过程、修改记录和最终结果 |

交互要求：

- 人工答案必须填写依据或无法确认说明。
- 提交处理前应做质量检查。
- 回流动作需记录处理人和回流目标。

## 5.11 模型与阈值配置页

路由：

```text
/admin/model-config
```

## 6. 后台管理关键交互边界

- 后台所有页面必须按角色和数据权限控制可见范围。
- 政策发布、权限变更、模型发布、Prompt 发布、阈值变更必须二次确认。
- 已发布政策的修改应进入版本管理，不应静默覆盖线上版本。
- 审计日志只读，不能删除或修改。
- 敏感材料、企业画像、模型输入输出需要按权限脱敏展示。
- 人工兜底答案必须包含依据或无法确认说明。
- Prompt、模型路由和阈值配置必须支持版本、灰度、回滚和审计。
- 看板指标必须提供口径说明，避免误读。
