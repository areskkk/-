# 企业端前端需求文档

## 1. 文档说明

本文档用于说明企业端前端的页面范围、路由位置、页面模块和模块功能点。企业端面向企业经办人，核心目标是让企业完成政策咨询、企业画像维护、政策匹配、资格预判、申报提交、进度查看和补正处理。

本文档只描述企业端业务页面。统一登录、注册、无权限页、404 页等公共能力可由公共入口统一承接，企业用户登录后进入企业端。

## 2. 企业端整体信息架构

企业端统一路由前缀：

```text
/enterprise
```

企业端主流程：

```text
企业绑定 → 企业画像 → 政策问答 / 政策推荐 → 资格预判 → 新建申报 → 上传材料 → 提交申报 → 查看进度 → 补正处理
```

## 3. 路由总览

| 路由 | 页面名称 | 页面用途 |
| --- | --- | --- |
| `/enterprise/bind` | 企业绑定页 | 企业用户首次进入后绑定企业主体，提交企业名称、统一社会信用代码、营业执照和经办人信息 |
| `/enterprise/dashboard` | 企业工作台 | 汇总展示推荐政策、待办事项、申报进度、补正提醒和政策问答快捷入口 |
| `/enterprise/profile` | 企业画像页 | 维护企业基本信息、经营数据、发展状态和扩展画像字段 |
| `/enterprise/qa` | 政策问答页 | 企业通过自然语言咨询政策，系统返回答案、引用来源、适用条件和下一步操作 |
| `/enterprise/policies` | 政策推荐与政策列表页 | 展示可申报政策、政策筛选、匹配情况和资格预判入口 |
| `/enterprise/policies/:policyId` | 政策详情页 | 展示政策详情、申报条件、材料要求、办理流程、资格预判和申报入口 |
| `/enterprise/applications` | 我的申报列表页 | 展示企业所有申报记录、状态、提交时间和可执行操作 |
| `/enterprise/applications/new` | 新建申报页 | 选择政策、填写申报信息、上传材料、确认 OCR 结果并提交申报 |
| `/enterprise/applications/:id` | 申报详情页 | 展示申报基础信息、状态时间线、材料、审核意见和可执行操作 |
| `/enterprise/supplements/:id` | 补正处理页 | 展示补正原因、缺失材料、截止时间，支持补充材料并重新提交 |
| `/enterprise/notifications` | 消息通知页 | 展示补正通知、审核结果、政策更新、系统提醒等消息 |
| `/enterprise/settings` | 账号与企业设置页 | 维护个人账号信息、当前企业、企业成员和基础偏好设置 |

## 4. 通用布局与公共模块

### 4.1 企业端应用布局

适用路由：`/enterprise/*`

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 顶部导航栏 | 展示系统名称、当前企业、消息入口、用户菜单、退出登录 |
| 侧边菜单 / 移动端底部导航 | 展示工作台、政策问答、企业画像、政策推荐、我的申报等入口 |
| 当前企业切换 | 当账号绑定多个企业时，允许切换当前操作企业 |
| 全局消息提醒 | 对补正、审核结果、系统异常进行全局提示 |
| 权限控制 | 根据用户身份和企业绑定状态控制页面访问 |
| 加载与错误状态 | 所有页面统一展示加载中、空状态、无权限、接口异常和重试入口 |

### 4.2 企业上下文模块

适用页面：企业端全部业务页面。

功能点：

- 保存当前企业 `enterprise_id`。
- 在请求企业画像、申报、政策推荐、材料上传时自动携带当前企业上下文。
- 当前账号未绑定企业时，引导跳转至 `/enterprise/bind`。
- 企业绑定状态异常时，限制申报提交等关键操作。

### 4.3 响应式规则

企业端需要同时支持桌面 Web 和 H5 手机端。

| 场景 | 桌面端表现 | 手机端表现 |
| --- | --- | --- |
| 表格列表 | 表格 + 筛选栏 + 批量操作 | 卡片列表 + 顶部筛选抽屉 |
| 申报表单 | 左侧步骤条 + 右侧表单内容 | 单列分步表单 + 底部固定操作栏 |
| 材料上传 | 拖拽上传、文件选择 | 拍照、相册、文件选择 |
| 政策问答 | 左侧会话列表 + 右侧问答区域 | 单会话沉浸式聊天界面 |
| 申报详情 | 信息分区 + 时间线 | 折叠面板 + 状态卡片 |

## 5. 页面详细需求

## 5.1 企业绑定页

路由：

```text
/enterprise/bind
```

页面目标：企业用户首次使用系统时完成企业主体绑定，为画像维护、政策推荐和申报提交提供企业身份基础。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 绑定状态提示 | 展示当前账号是否已绑定企业；已绑定时展示企业名称、信用代码、绑定状态 |
| 企业基础信息表单 | 输入企业名称、统一社会信用代码、注册地址等基础信息 |
| 营业执照上传 | 上传营业执照图片或 PDF；展示上传进度、文件名、大小、失败原因 |
| 经办人信息 | 输入经办人姓名、手机号、职位、授权说明 |
| 自动校验提示 | 展示企业名称与证照识别结果是否一致、信用代码格式是否正确 |
| 提交绑定申请 | 提交企业绑定信息；提交后展示处理中、已通过、被驳回等状态 |
| 驳回原因展示 | 当绑定失败时展示驳回原因，并允许修改后重新提交 |

交互要求：

- 未完成企业绑定的用户访问企业端核心页面时，应提示先完成绑定。
- 企业绑定提交后，页面应展示清晰的处理状态。
- 营业执照上传失败时，应保留已填写表单数据，允许重新上传。

## 5.2 企业工作台

路由：

```text
/enterprise/dashboard
```

页面目标：作为企业端首页，集中展示企业当前可办理事项、申报进展、补正提醒和政策咨询入口。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 企业概览卡片 | 展示当前企业名称、统一社会信用代码、企业画像完整度、当前申报总数 |
| 画像完善提醒 | 当关键画像字段缺失时，提示补全行业、营收、员工、纳税等字段 |
| 推荐政策区 | 展示推荐政策卡片，包括政策标题、主管部门、匹配度、申报截止时间、资格预判状态 |
| 待办事项区 | 展示草稿待提交、需补正、待确认 OCR、待完善画像等事项数量和入口 |
| 申报进度区 | 展示最近申报记录、当前状态、更新时间、进入详情按钮 |
| 补正提醒区 | 展示需补正事项、补正原因、截止时间、进入补正按钮 |
| 政策问答快捷入口 | 提供快速输入框，用户可直接输入问题并跳转至政策问答页 |
| 常用操作入口 | 提供新建申报、查看政策、维护画像、查看我的申报等快捷按钮 |

交互要求：

- 工作台数据应围绕当前企业展示。
- 推荐政策卡片应能直接进入政策详情、资格预判或新建申报。
- 补正提醒应突出截止时间和紧急程度。
- 当暂无推荐政策或暂无申报时，应展示可操作的空状态引导。

## 5.3 企业画像页

路由：

```text
/enterprise/profile
```

页面目标：采集和维护企业画像字段，支撑政策推荐、资格预判和申报材料预填。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 基础信息模块 | 展示和维护企业名称、统一社会信用代码、所属行业、注册地址等基础信息 |
| 经营数据模块 | 维护企业规模、营收、员工数、纳税额、出口额等字段 |
| 发展状态模块 | 维护技改状态、高新资质、研发投入、设备投入等字段 |
| 扩展画像字段 | 支持通过扩展 JSON 或动态表单展示政策相关补充字段 |
| 画像完整度 | 计算关键字段完整度，提示缺失字段及其影响 |
| 字段说明提示 | 对营收、纳税额、出口额等敏感字段展示单位、口径和用途说明 |
| 保存操作区 | 支持保存画像、重置未保存修改、展示保存成功或失败信息 |
| 画像更新时间 | 展示最近更新时间和更新人 |

交互要求：

- 表单保存前应进行格式校验。
- 金额类字段应明确单位。
- 画像更新只影响新申报，历史申报应继续使用提交时的画像快照。
- 当画像缺失影响资格预判时，应在页面内明确提示。

## 5.4 政策问答页

路由：

```text
/enterprise/qa
```

页面目标：企业通过自然语言咨询政策，系统返回可追溯、可解释、可继续操作的答案。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 会话列表 | 展示历史咨询会话、新建会话入口、会话标题、更新时间 |
| 问题输入区 | 支持文本输入、快捷问题、发送按钮；后续可扩展语音转文字 |
| 用户问题气泡 | 展示用户输入的问题内容和发送时间 |
| 答案卡片 | 展示简明答案、适用条件、注意事项和下一步操作 |
| 引用来源区 | 展示政策名称、条款片段、来源链接、引用可信度 |
| 适用条件区 | 展示该政策对行业、规模、区域、时间、材料等条件要求 |
| 下一步操作区 | 提供完善画像、资格预判、查看政策详情、开始申报等按钮 |
| 追问建议 | 根据当前问题生成推荐追问，如“我是否符合条件”“需要哪些材料” |
| 低置信度提示 | 当无引用、低置信度、规则冲突时，提示不能作为正式结论 |
| 人工咨询入口 | 对无法确认的问题提供转人工或提交咨询的入口 |

交互要求：

- 无引用答案不能作为正式政策结论展示。
- 答案必须突出来源依据，引用区域可展开查看原文片段。
- 当回答需要补充企业画像时，应引导用户跳转至画像页补充字段。
- 问答页中的“开始申报”必须先经过政策详情或资格预判确认。

## 5.5 政策推荐与政策列表页

路由：

```text
/enterprise/policies
```

页面目标：展示企业可查看和可申报的政策，支持筛选、查看匹配情况和发起资格预判。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 筛选区 | 支持按关键词、主管部门、政策类型、申报状态、截止时间筛选 |
| 推荐排序 | 可按匹配度、截止时间、发布时间、热度等排序 |
| 政策卡片 / 列表 | 展示政策标题、主管部门、政策摘要、有效期、申报截止时间 |
| 匹配度标识 | 展示推荐分、匹配原因、缺失画像字段提示 |
| 资格预判标识 | 展示符合、不符合、待补充、需人工复核等状态 |
| 操作按钮 | 支持查看详情、资格预判、开始申报、收藏政策 |
| 空状态引导 | 无匹配政策时，引导补充画像或发起政策问答 |

交互要求：

- 政策卡片应突出申报截止时间。
- 当企业画像不完整导致推荐不准确时，应明确提示。
- 列表页可直接触发资格预判，但详细解释应在政策详情或预判结果中展示。

## 5.6 政策详情页

路由：

```text
/enterprise/policies/:policyId
```

页面目标：展示单个政策的完整信息，帮助企业判断是否适合申报。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 政策头部信息 | 展示政策标题、主管部门、政策级别、版本、生效时间、截止时间、状态 |
| 政策摘要 | 展示政策核心内容、支持方向、适用企业类型 |
| 适用对象 | 展示行业、区域、规模、营收、员工、纳税、出口、技改等条件 |
| 申报条件 | 结构化展示硬性条件、可补充条件、需人工判断条件 |
| 材料要求 | 展示必填材料、选填材料、材料有效期、可复用说明 |
| 办理流程 | 展示申报、预审、审核、补正、结果通知等流程 |
| 政策原文 | 支持查看政策原文、条款、附件和来源链接 |
| 资格预判区 | 发起资格预判，展示结果、原因、缺失字段和缺失材料 |
| 申报入口 | 符合或可补充时，提供开始申报按钮 |
| 咨询入口 | 提供针对当前政策的问答入口 |

交互要求：

- 政策过期或废止时，应禁止新建申报，并展示原因。
- 材料要求应明确是否必填、是否允许本次申报内复用。
- 资格预判结果应可展开查看条件级判断依据。

## 5.7 我的申报列表页

路由：

```text
/enterprise/applications
```

页面目标：集中展示当前企业的所有申报记录和状态，支持继续处理草稿、查看进度和进入补正。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 状态筛选 | 支持按草稿、已提交、预审中、审核中、需补正、已通过、不通过、已撤回等状态筛选 |
| 时间筛选 | 支持按创建时间、提交时间、更新时间筛选 |
| 申报列表 | 展示申报编号、政策名称、申报类型、当前状态、提交时间、更新时间 |
| 状态标签 | 使用统一颜色和文案展示申报状态 |
| 操作按钮 | 根据状态展示继续编辑、查看详情、撤回、补正、查看结果等操作 |
| 多政策子项摘要 | 多政策申报时展示各政策子项状态概览 |
| 空状态引导 | 无申报时引导查看政策或新建申报 |

交互要求：

- 草稿状态允许继续编辑和提交。
- 已提交状态允许查看和撤回。
- 需补正状态应突出补正入口和截止时间。
- 已通过或不通过状态应提供结果查看入口。

## 5.8 新建申报页

路由：

```text
/enterprise/applications/new
```

页面目标：引导企业完成政策选择、资格预判、申报表单填写、材料上传、OCR 确认和申报提交。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 步骤导航 | 展示选择政策、资格预判、填写信息、上传材料、确认提交等步骤 |
| 政策选择器 | 支持从推荐政策或政策列表中选择申报政策；后续支持同次选择多个政策 |
| 资格预判卡片 | 展示预判结果、命中条件、缺失字段、缺失材料、需人工复核原因 |
| 动态申报表单 | 根据政策结构化 schema 生成申报字段，支持画像预填和手动修改 |
| 材料清单 | 展示当前政策所需材料、是否必填、是否已上传、是否过期 |
| 材料上传组件 | 支持上传图片、PDF、Word 等材料；展示上传进度、失败重试和文件信息 |
| OCR 结果确认 | 展示识别字段、置信度、异常提示，支持人工确认或修改 |
| 材料复用提示 | 多个政策需要同一材料时，提示本次申报内复用 |
| 草稿保存 | 支持保存当前填写进度，手机端可分步保存 |
| 提交确认 | 提交前展示企业画像快照、政策、材料、承诺确认和提交按钮 |

交互要求：

- 手机端必须采用分步单列，不得简单缩放桌面页面。
- 上传失败时应保留草稿和已填写内容。
- 提交前必须展示申报摘要和材料完整性检查结果。
- 低置信度 OCR 字段应要求用户确认。
- 政策已过期、材料缺失、画像关键字段缺失时，应阻止或提示无法提交。

## 5.9 申报详情页

路由：

```text
/enterprise/applications/:id
```

页面目标：展示申报全生命周期信息，让企业了解当前状态、材料、审核意见和下一步操作。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 申报基础信息 | 展示申报编号、企业名称、政策名称、申报类型、创建时间、提交时间 |
| 当前状态卡片 | 展示当前状态、状态说明、下一步操作建议 |
| 状态时间线 | 展示草稿、提交、预审、审核、补正、补正提交、通过、驳回等节点 |
| 政策子项状态 | 多政策申报时展示每个政策子项的状态、审核结果、补正要求 |
| 企业画像快照 | 展示提交时冻结的企业画像关键信息 |
| 材料列表 | 展示已上传材料、材料类型、上传时间、OCR 状态、预览入口 |
| 审核意见 | 展示审核结论、驳回原因、补正原因、审批意见 |
| 操作区 | 根据状态展示撤回、进入补正、再次咨询、查看政策等操作 |
| 审计与记录 | 展示关键操作记录，如提交、撤回、补正提交、审核结果更新时间 |

交互要求：

- 不同状态下展示不同操作按钮，避免无效操作。
- 需补正状态应明显展示补正原因和截止时间。
- 已撤回、已通过、已驳回状态应以只读方式展示。
- 材料预览不可用时，应展示文件基础信息和不可预览原因。

## 5.10 补正处理页

路由：

```text
/enterprise/supplements/:id
```

页面目标：企业根据政府端补正要求补充材料或修改信息，并重新提交。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 补正说明 | 展示补正原因、发起部门、发起时间、截止时间 |
| 补正材料清单 | 展示需要补充或替换的材料、材料要求、示例说明 |
| 问题字段列表 | 展示需要修改的表单字段、原值、问题原因、建议修改方式 |
| 材料上传区 | 上传补正材料，支持替换旧文件、补充新文件、失败重试 |
| OCR 复核区 | 对补正材料触发 OCR 后展示识别结果和异常提示 |
| 补正说明输入 | 企业可填写补正说明或备注 |
| 提交补正按钮 | 提交补正材料和说明，提交后状态更新 |
| 历史补正记录 | 展示历史补正要求和提交记录 |

交互要求：

- 补正页只允许处理处于需补正状态的申报。
- 截止时间应固定展示，临近截止时使用明显提醒。
- 提交补正前应校验必补材料是否已上传。
- 提交成功后跳转申报详情页并展示状态变化。

## 5.11 消息通知页

路由：

```text
/enterprise/notifications
```

页面目标：集中展示企业用户相关通知，避免错过补正、审核结果和政策更新。

页面模块：

| 模块 | 功能点 |
| --- | --- |
| 消息分类 | 按补正通知、审核结果、政策更新、系统通知分类 |
| 消息列表 | 展示标题、摘要、时间、已读状态、关联业务对象 |
| 消息详情 | 展示完整通知内容和关联操作入口 |

交互要求：

- 未读补正通知应在顶部导航中显示数量。
- 关键通知不能只依赖弹窗，应在消息中心可追溯。

## 6. 企业端状态展示规则

| 后端状态 | 企业端文案 | 页面表现 | 可执行操作 |
| --- | --- | --- | --- |
| `draft` | 草稿 | 灰色标签 | 编辑、提交、删除 |
| `submitted` | 已提交 | 蓝色标签 | 查看、撤回 |
| `pre_reviewing` | 预审中 | 蓝色进度标签 | 查看 |
| `reviewing` | 审核中 | 蓝色进度标签 | 查看 |
| `need_supplement` | 需补正 | 橙色警示标签 | 补充材料、提交补正 |
| `resubmitted` | 已补正 | 蓝色标签 | 查看 |
| `manual_review` | 人工复核中 | 紫色标签 | 查看、咨询 |
| `approved` | 已通过 | 绿色标签 | 查看结果 |
| `rejected` | 不通过 | 红色标签 | 查看原因 |
| `withdrawn` | 已撤回 | 灰色标签 | 查看 |
| `timeout_closed` | 超时关闭 | 灰色标签 | 查看归档 |

## 7. 企业端关键交互边界

- 企业未绑定时，不允许新建申报。
- 企业画像缺失关键字段时，应允许继续浏览政策，但资格预判需提示结果可能不完整。
- 政策问答无引用或低置信度时，不展示为正式结论。
- 材料复用仅限同一次申报内，不做长期材料库。
- 申报提交时应冻结企业画像快照。
- 已提交申报不能直接修改，只能撤回或按补正要求提交补正。
- 手机端申报必须支持分步保存、上传失败重试和底部固定操作栏。
