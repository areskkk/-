# 数据模型设计

> 适用范围：南康助企宝 MVP  
> 依据：`PRD改进.md`、四份 PRD、prd-planner 访谈记录  
> 口径：MVP 自建账号；本期不对接亲清赣商/惠企通；材料只在本次申报内复用。

## 1. ER 关系

```text
Enterprise 1 ── N EnterpriseAccount N ── 1 User
Enterprise 1 ── N EnterpriseProfileSnapshot
Enterprise 1 ── N Application 1 ── N ApplicationPolicyItem N ── 1 Policy
Application 1 ── N Material
Policy 1 ── N PolicyCondition
Policy 1 ── N PolicyMaterialRequirement
ApplicationPolicyItem 1 ── N ReviewRecord
User N ── 1 Department
User N ── N Role N ── N Permission
AnyEntity 1 ── N AuditLog
```

## 3. 状态枚举

| 枚举 | 值 |
| --- | --- |
| application_status | draft、submitted、pre_reviewing、reviewing、need_supplement、resubmitted、manual_review、approved、rejected、withdrawn、timeout_closed、archived |

| fallback_status | pending、processing、resolved、closed |

## 4. 关键约束

- 多政策申报：一个 `Application` 下有多个 `ApplicationPolicyItem`，政府端以子项为审核单元。
- 材料复用：只在同一 `Application` 内复用，不跨历史申报复用。
- 企业画像：申报提交时生成快照，历史申报不受后续画像修改影响。
- 企业绑定：Agent 可对低风险绑定自动通过，必须写入 `AuditLog` 和审核记录。
- 政策版本：已提交申报引用提交时政策版本；政策废止后进入人工复核。
