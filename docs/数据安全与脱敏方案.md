# 数据安全与脱敏方案

## 1. 数据分级

| 等级 | 数据 | 处理要求 |
| --- | --- | --- |
| L1 公开 | 政策原文、公开办事指南 | 可公开展示 |
| L2 内部 | 政策结构化规则、运营指标、权限配置 | 角色授权 |
| L3 企业敏感 | 企业画像、申报材料、联系方式、财税信息 | 加密、脱敏、最小权限 |
| L4 审批敏感 | 审核意见、风险标记、模型 trace | 严格授权、全量审计 |

## 2. 脱敏规则

| 字段 | 展示规则 |
| --- | --- |
| 手机号 | 138****1234 |
| 统一社会信用代码 | 前 4 后 4，中间隐藏 |
| 身份证号 | 前 3 后 4，中间隐藏 |
| 银行账号 | 后 4 位可见 |
| 纳税额/合同金额 | 非授权角色仅展示区间 |

## 3. 日志要求

- 日志不得记录完整身份证、银行账号、合同全文。
- 模型 prompt 和 trace 只保留必要上下文。
- 审计日志保留 actor、action、target、time、trace_id。

## 4. 权限要求

- 默认最小权限。
- 政府人员只可查看授权事项和材料字段。
- 后台操作高敏配置需记录变更原因。
