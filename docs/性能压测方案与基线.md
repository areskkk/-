# 性能压测方案与基线

## 1. MVP 基线

| 场景 | 目标 |
| --- | --- |
| 政策问答整体响应 | P95 <= 5s |
| 缓存命中简单问答 | P95 <= 3s |
| 检索召回 | P95 <= 1.5s |
| 资格预判 | P95 <= 8s |
| OCR 单页处理 | P95 <= 10s，可异步 |
| 申报提交 | P95 <= 3s，不含异步 OCR |
| 审核台列表 | P95 <= 2s |
| 页面首屏 | LCP <= 2.5s |

## 2. 并发模型

| 用户 | MVP 压测并发 |
| --- | --- |
| 企业用户 | 100 |
| 政府人员 | 50 |
| 后台管理员 | 10 |

## 3. 压测场景

1. 政策问答连续提问。
2. 政策检索和推荐列表。
3. 多政策资格预判。
4. 手机端申报提交。
5. 材料上传和 OCR 异步任务。
6. 政府审核任务列表和详情。
7. 后台政策导入和结构化保存。

## 4. 验收

- 达到基线指标。
- 高峰压测无数据错乱、重复提交、权限越权。
- OCR 和 Agent 异步任务积压可观测。
- 错误率低于 1%。
